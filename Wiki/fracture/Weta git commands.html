Interacting with the code repository

The code is stored on Weta's machine. Access is via ssh/rsa.

Setting up access to the repository

  • On your machine run
    ssh-keygen -t rsa -C "your_email@youremail.com"

  • The command will generate a public/private key pair and ask you for a passphrase which you will have to save for future reference. Use something you can easily remember, e.g., my weta entry code.
  • More detail about the process can be found at https://help.github.com/articles/generating-ssh-keys
  • Windows users should use the additional instructions at http://stackoverflow.com/questions/11771378/ssh-keygen-is-not-recognized-as-an-internal-or-external-command
  • e-mail your id_rsa.pub file to Andreas at <asoderstrom@wetafx.co.nz>
  • Andreas will do some magic on his end that will allow you access to the machine. Wait for his reply.

Copying the repository to your machine

  • You will have to install git on your machine. For Linux users git should be available directly in Ubuntu/Debian/Redhat etc. precompiled binary form. If you run into any problems let us know. Windows users should see http://msysgit.github.com/
  • To get the code run
    git clone gitosis@git.wetafx.co.nz:ParticleFracturing

    If it asks you for a passphrase enter the stuff that you used to create your ssh key.

Interacting with the repository

The basic commands you will need are
  • To add new files to the repository
    git add <filename>
  • To commit changes to your local branch
    git commit

    or
    git commit -a
  • To push your changes to the main repository
    git push
  • To update your local repository
    git pull
http://rogerdudler.github.com/git-guide/ is a nice webpage containing commands that you are likely to be interested in.
