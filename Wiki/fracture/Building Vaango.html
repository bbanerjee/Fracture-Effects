• When you get your repository code, follow these steps:
    ls
to bring the list of directories. Now you are able to see a new directory enabled by your code (ParticleFracturing) enter this directory:
  • cd ParticleFracturing/
  • to see inside again:
    ls
  • Now you can see the directory named: Vaango. Enter it by this command:
    cd Vaango/

followed by:
  • ls
you can see a file named "src" you can see inside it by the same command: "cd src/"

  • now you need to build a new directory for yourself as opt:
    mkdir opt
followed by:
  • ls
this file is empty now but you need to bring all the needed files and programs from other directories. To do that you need to follow these steps when you are in the opt directory:
    • first
  • cmake ../src
      • ../ refers to file-name in parent directory. So if you are in opt, by ../ you will refer to the parent directory, which is Vaango, where exactly src exists.
      • By the way, if you are currently in Vaango you just need to print ./src cause ./ refers to file-name in current directory
By this you make a cmake file in src directory which would be the connection between your newborn directory: "opt" and src. At the end with the command "make" you would compile/recompile and update automatically all the needed files from src into opt through cmake.

  • Here you probably need to install Cmake to control the software compilation process. To do that:
    sudo apt-get install cmake
use the following command if needed:
  • sudo apt-get-f install
This command can omit any packages in order to permit APT to deduce a likely solution. It fixes any breaks happening during the installation.
  • you also need to have gfortran, a free Fortran compiler:
    sudo apt-get install gfortran
  • and libxml2:
    sudo apt-get install libxml2
  • A C Compiler known as gcc:
    sudo apt-get install gcc
and:
  • sudo apt-get install mpi-default-dev
  • sudo apt-get install libxml2-dev
  • You will also need to install the development version of zlib.
      sudo apt-get install zlib1g zlib1g-dev
  • In order for related code like BrMPM etc. to work, you will also need the VTK libraries. Use
      sudo apt-get install libvtk5-dev
  sudo apt-get install python-vtk tcl-vtk libvtk-java libvtk5-qt4-dev
  • In some older versions of Vaango libpcl is used read point data file. In order to get PCL(The Point Cloud Library) you need to copy these three commands:
    sudo add-apt-repository ppa:v-launchpad-jochen-sprickerhof-de/pcl
sudo apt-get update
sudo apt-get install libpcl-all
    • Print each line once at a time otherwise it just runs the first line and the PCL wont be installed.
  • If you need other packages try:
    apt-cache search <package-name> | more
this command Performs a full text search on all available package lists.

  • Now you can make the cmake file:

  • cmake ../src

    • Then you need to compile the needed files from src. So enter your opt directory and then type : "make"
It should be in this form:
  • ...:~/ParticleFracturing/Vaango/opt$ make
After this compilation you have all the files needed in your "opt" directory.


  • Go back to Vangoo again. Now here you have src and opt and with the below command you can make another directory as "runs"
    mkdir runs
cd runs
  • It is empty now but you can make a link from it to the written files in other directories (for example: src/StandAlone/inputs/MPM) as "inputs":
    ln -s ../src/StandAlone/inputs/MPM inputs
followed by
  • ls -ls
to see the list in long format with file size inside that link.

  • Now with the following command you can see and edit the written program in each files (like the HTML document: billet.ups)
    vi inputs/billet.ups
      • "vi" is an editor. You can use others like gedit,...
  • In order to run the program you need to make another link to a source code (vaango / a C++ source code):
    ln -s ../opt/StandAlone/vaango vaango
  • Now run it in your current directory ("runs") by:
    ln -s ./vaango inputs/billet.ups
