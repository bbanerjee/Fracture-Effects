Here in Matiti directory we have two directorie: src (as source) and Triangle (which gives us a library needed to make triangles on a surface)

In order to run the programs in the source → (~/ParticleFracturing/Matiti/src/input_files), we have to:
  1. create two directories:
    • opt ← optimized build (it is much faster to run)
    mkdir opt
    • dbg ← debug build (needs time to find the bugs and debug it)
    mkdir dbg
  2. create one directory:
    • runs ← where you save the out-put-data as VTK format files in order to visualise them by visit.
    mkdir runs
So now you have:





  • I. First of all, you need to make the Triangle Library:
  •  ~/ParticleFracturing/Matiti$ cd Triangle
 ~/ParticleFracturing/Matiti/Triangle$ ls
Here again you have src as source and need to make an opt directory. Moreover you can see a READ me file which gives you information on how to compile the package.
In order to compile:
  •  ~/ParticleFracturing/Matiti/Triangle$ cd opt
 ~/ParticleFracturing/Matiti/Triangle/opt$ ls
 ~/ParticleFracturing/Matiti/Triangle/opt$ cmake ..
SO by this you have made a makefile ==> generators are built.
Now if you print:
  •  ~/ParticleFracturing/Matiti/Triangle/opt$ make
you will make a library called libTriangle++.so* and other files out of the files in src.
In order to see all the latest files created by the above command, just type:
  •  ~/ParticleFracturing/Matiti/Triangle/opt$ ls -lt




  • II. Go back to Matiti level. Here again you can see our source (src). You put all the optimized files built out of src in the opt.

  •  ~/ParticleFracturing/Matiti/src$ ls

Here you can see that there is a cmakelist.txt which has listed all the needed input files.

Go to opt
  •  ~/ParticleFracturing/Matiti$ cd opt
 ~/ParticleFracturing/Matiti/opt$ cmake ../src
 ~/ParticleFracturing/Matiti/opt$ ls
 ~/ParticleFracturing/Matiti/opt$ make
 ~/ParticleFracturing/Matiti/opt$ ls -lt    → latest files created would be shown

Now that all the optimized files are brought into opt, it is time to run your desired program. Go to runs directory and do:
  •  ~/ParticleFracturing/Matiti$ cd runs
 ~/ParticleFracturing/Matiti/runs$ cd Box
 ~/ParticleFracturing/Matiti/runs/Box$ ls

As you may see, all the old out-put-data and the program is here. In order to run any other programs you need to bring that particular program (from the source directory → ~/ParticleFracturing/Matiti/src/input_files/) into this directory. Here we have just test_box_crack.van. In order to run this program type:
  •  ~/ParticleFracturing/Matiti/runs/Box$ ~/ParticleFracturing/Matiti/opt/text_peri test_box_crack.van
  • * test_peri is an executable file which is brought into the opt from the src

NOTE You may get an ERROR while you are running the program saying that the address does not match. This happens when you bring the opt directory along with the other directories when you git pull. So in order to make everything right, delete the existing opt directory and make a new one:
    •  rm -r opt
 mkdir opt
and go through all the above steps. Now it will do the job!!

Now after running the program and having all the out-put data:
  •  ~/ParticleFracturing/Matiti/runs/Box$ ls

  •  test_box_crack    →   #the name of the out-put-data directory after running the program for the first time.
 test_box_crack01  →   #the name of the out-put-data directory after running the program for the second time.
 test_box_crack02  →   #the name of the out-put-data directory after running the program for the third time.
 .                      .
 .                      .
 .                      .


To visualise the out-put-data with visIt program do as below:
  •  ~/ParticleFracturing/Matiti/runs/Box$ visit  →   #visit is a short-cut name for calling the "visIt" program.

When the visit windows are opened, you can see the "open" icon. Click it and from the new window choose your out-put data directory on the left box. By clicking on the data directory on the left box all the data will be appeared on the right box. The data consists of two kinds of data: the first one is your points' data and if you scroll down you can find the second kind of data which belongs to the domain.

  • first click on the point data (when the scroll is up):
    then from them tool bar, on the left, choose:
     Add → Pseudocolor → displacement_magnitude → Draw

  • So your points will be shown as dots. In order to give them volume you can choose "plot Attributes" from the tool bar and:
  •  plot atts → Pseudocolor → Point type: sphere, 10 pixel → apply

  • Up to now you have brought your points without any border or domain. In order to bring the domain again click on the "open" icon and in the right hand box scroll down to the domain data and click on it to choose the data. Then again add the properties so you can visualise the domain as well:
     Add → Mesh → choose (Yes) → Draw

Now your object (which is a box for the above program) is located in a box as its domain.

Now if you click on the "play" icon you can see the movement of your object in the domain.
